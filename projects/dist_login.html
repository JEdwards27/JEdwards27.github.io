About the project

overview

Project aims
  • Obfuscate security data through distribution
  • Increase factors of authentication without increasing complexity
  • Felxibility for integration into a number of use cases
At a glance
  • Login key is generated from user credentials and datestamp
  • Key is fingerprinted, this fingerprint is stored in SQL database.
  • Key is split up and distributed amongst several SQL databases
  • One part of the key is placed into a USB drive
  • Login re-compiles split-up key and fingerprints it
  • If the fingerprint matches the stored print, user is authenticated

Results

performance

UX Timings
  • Project was adapted for use with ssh as a login mechanism
  • Other popular MFA methods for ssh were set-up for comparison
  • 3 Users were timed from starting connection, to disconnection
  • Note: The performance of the proposed system were hindered by the 'send key' stage of the login process. This was due to time constraints not allowing for automation of this function.

security

Authentication Factors
  • Project has 4 factors of authentication
  • These are: Password, USB drive, last group signout & certificates
  • These are 2 things the user knows, and two things the user has
Further development

  • > An account recovery function
  • > Testing out further distribution of security data using 8 SQL databases implemented with logical groups
  • > Integration with YubiKey instead of USB drive
  • > Develop a user interface to further increase usability and user friendliness
